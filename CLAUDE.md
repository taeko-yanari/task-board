## Git 運用ルール

### 基本方針
- コードを変更するたびに、必ず GitHub にプッシュする
- コミットは意味のある単位でまとめ、メッセージは変更の「なぜ」を端的に記述する

### ワークフロー
1. ファイルを変更する
2. 変更内容を確認する: `git diff`
3. 関連ファイルをステージングする: `git add <files>`
4. コミットする: `git commit -m "..."`
5. **必ず** リモートにプッシュする: `git push`

### コミットメッセージ規約
- 英語または日本語で記述
- 先頭に種別プレフィックスを付ける:
  - `feat:` 新機能
  - `fix:` バグ修正
  - `refactor:` リファクタリング
  - `docs:` ドキュメント
  - `test:` テスト
  - `chore:` ビルド・設定変更

### 禁止事項
- `git push --force` は main/master ブランチへ使用しない
- `--no-verify` でフックをスキップしない
- 未確認の破壊的操作（`reset --hard`, `clean -f` 等）は実行しない

## コーディング規約

- コメントは「なぜ」が自明でない場合にのみ追加する
- 不要な抽象化・将来の要件への対応は行わない
- セキュリティ: コマンドインジェクション・XSS・SQLインジェクション等を避ける
